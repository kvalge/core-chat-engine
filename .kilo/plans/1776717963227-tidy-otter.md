## Feature Analysis

### Short Name Generation
- **Proposed**: `core-chat-engine` (application name is descriptive enough)

### Key Concepts Extracted
1. **Multi-turn chat** - Conversation state, message history, streaming responses
2. **OpenAI-compatible API** - /v1/chat/completions endpoint, Ollama support
3. **Tool use loop** - LLM-driven tool calls (date/time, web search, todo list, calendar)
4. **Projects** - Custom system prompts, default models, per-project tool config
5. **File upload** - Text, PDF, images with vision model support
6. **Backend configuration UI** - Add/edit/remove API backends, test connection

### Scope Assessment
This is a **full-stack web application** with multiple interdependent features. Per Constitution principle, it requires:
- Backend: Python FastAPI with async REST API
- Frontend: React 19, TypeScript, Vite
- Local LLM: Ollama (llama3.2, mistral)
- Streaming: SSE hand-rolled
